Robot Arm Control Interface

Following the success of the Android application I developed to control my robot arm, I developed a robot arm control desktop application as a project to learn Python and PyQt5 (the Python binding of the popular Qt application framework). The application had an easy to use graphical user interface, and builds on the functionality of the original app. The code can be found on the GitHub repository.

The application communicated via a USB cable to an Arduino with GRBL (open-source CNC control software) installed. Serial G-code commands could then be sent from the program to the Arduino to move the robot arms motors.

The diagram below highlights the different sections of the interface.

The main part of the program is the jog controls, which are split into arm and end effector controls. The arm controls include buttons for jogging each axis, and sliders to control how much the arm moves for each button press. Similarly, the end effector controls include buttons to jog the angle of the end effector, and open/close the gripper, with adjustment sliders.

The program is equipped with inverse kinematics, which is why the arm control buttons are labelled forward/backward, left/right, up/down and so on, rather than having button for each specific motor. When the forward button is pressed, the inverse kinematic equations calculate the required angles for each axis to move the end effector straight forward by the specified amount. The pre-configured GRBL software running on the Arduino then calculates how many steps the NEMA-17 stepper motors need to move from their current position to reach the new position, taking into account the planetary gear box ratios.

A 3D graph is displayed in the top left of the interface, using Matplotlib, which uses graphical lines to represent the current position of each arm axis. This can be used to simulate the position of the robot arm without having the physical robot arm connected.

On top right side of the interface, the serial interface can be configured by selecting the COM port of the Arduino, and functions such as calibrating (via limit switches) and homing can be executed.

In the centre of the interface, the current position coordinates of each robot arm axis are displayed, and can be edited via keyboard input to set an exact position.

The bottom half of the interface is responsible for configuring user-defined position sequences that the arm can perform. The user can either create sequence file, or open an existing one in the form of a standard .txt file, containing lines of G-code commands.

The user can the jog the arm using the control buttons to its desired position, before pressing the Add New button, which will add the current position to the list. The user can then move the arm to the next position, press Add New and so on. There are also options to overwrite a position on the list by selecting a position on the list before pressing the Replace button. The Remove button will delete the selected list item, obviously, and the Move Up and Move Down buttons will shift the selected list item up and down the list.

The speed dial determines the speed at arm moves to each position, and is saved to each position on the list, allowing some movements to be slow, and others rapid.

Once a sequence has been programmed, the Play and Stop buttons can be pressed to make the arm sequentially move to each position on the list. If the repeat toggle is enabled, the arm will continuously repeat the same movements until manually stopped – useful to automating tasks.

Overall, the project was a success, and as my first exposure to object orientated programming, I learnt a lot about the features of Python, using Classes and the different widgets/layouts available in PyQt5.

The knowledge and experience obtained during this little project were essential for developing the main pilot control interface for an underwater remotely operated vehicle competition.
